What's FoOlReader?

FoOlReader is a modern image browser, specifically meant to be used with comics.
It's special because it doesn't need page reloads just to change the visualized image, unlike basically every other comic reader.

It also automatizes a lot of necessary work to display lists of available comics, and has custom themes. It's very easy to install, it works on almost every server, and gets your readers to love your website more.

It uses the following technologies to work: PHP, JavaScript, jQuery, HTML, Woxxy's brain.

How to install

Just extract the files into a folder with any name you'd like. In example:
/var/www/comicreader/
and put the files in there.

Then open the file
/var/www/comicreader/settings.php
and add what it asks for.

How to add comics

Create a folder called "content". Like this:
/var/www/comicreader/content/
This folder will contain everything related to your comics.

Every comic will take name from its folder:
/var/www/comicreader/content/Sora no Otoshimono/

Same story to add a chapter:
/var/www/comicreader/content/Sora no Otoshimono/chapter 45/
or any chapter name you'd like:
/var/www/comicreader/content/Sora no Otoshimono/omake 3/
and the images:
/var/www/comicreader/content/Sora no Otoshimono/chapter 45/03.png


How to add comics' descriptions and thumbnails

While it will work even without them, you better take some time to add these.

To add a description, write a file like this:
/var/www/comicreader/content/Sora no Otoshimono/description.txt
You can use HTML! I suggest using <small></small> in case it's too much text to be added.

To add thumbnails, add a small picture like this:
/var/www/comicreader/content/Sora no Otoshimono/thumb.png
or in jpeg:
/var/www/comicreader/content/Sora no Otoshimono/thumb.jpg


How to enable advertisings

There's a
/var/www/comicreader/ads/
folder, with 3 files. Each filename has the position of the ad.

Avoid editing the first line, and add the code for a 728px*90px ad after the second line.

How to update

Save on your HD the settings.php and the
/var/www/comicreader/ads/
folder, just so you have some reference and can copy/paste.

Grab the latest version at FoOlRulez.

Delete all the folders beside
/var/www/comicreader/content/
that stores your comics.

Upload all the new files from the new version.

Update manually
/var/www/comicreader/settings.php
and the
/var/www/comicreader/ads/
folder.

Suggestions

There's always an extra bunch of stuff you don't know. Here's some stuff that might interest you.

  • About the server

    The kind of server you should prefer using is a VPS (Virtual Private Server), that is a virtualized computer inside a bigger computer, loosely said. I suggest never going for servers that offer "infinite bandwidth" and "infinite space", because that's a lie. It rather means "it's so slow you will never use so much bandwidth".

    I get often asked on what I host, because it's impressively fast. My host is linode.com, capable of pushing over 30mbit/s for the cheapest plan. That's wide enough to fill the bandwidth of the client. I run projects much heavier than this comics reader, and it keeps up like a charm. I suggest jumping on their servers if you are in need of a hosting that works great.

    Furthermore, I use Nginx with PHP-FPM as webserver, on Debian 5. Apache is what people use because it's most popular, but serious business when you need to save money requires Nginx. It will perform several times faster than Apache. If you're familiar with linux or can at least follow tutorials, google about it.

    What's nice, is that linode has stackscripts, so even setupping Ubuntu with Nginx should be a click away.

    End of shameless linode advertising. Sorry but you guys asked for it.

  • About the bandwidth

    My group is not small, but we use less than 200gb per month, normally, and we also provide direct downloads. It's pretty hard to go over that much bandwidth use, unless your comics are way too popular.

  • For developers and designers

    The script has a theme system. All themes work like "child themes". You just need to create a new folder with:
    /var/www/comicreader/themes/yourthemename/style.css
    All other files will be automatically imported, unless you create substitutes in the same folder.

    I'll provide in-depth documentation for this separately. If you're in a hurry, you can check out the comments in the file
    /var/www/comicreader/functions/functions.php
    that contains the core of FoOlReader.

  • Why do I always say comics and not manga?

    Because in the code of FoOlReader there's always written comics. Precisely, you can't say "mangas", and I needed a plural so the names of the functions would be easier to understand. So I used comic instead. And I'll keep doing it.
